0. 복습
1) sql 실행순서
	FROM > WHERE > GROUP BY > HAVING > SELECT > ORDER BY

2) 집계함수
	SQL에서 데이터를 요약하거나 통계정보를 계산하는데 사용되는함수
	그룹화된 데이터나 전체 데이터에 대해 특정연산을 수행해서 하나의 행을반환한다.
	행이 다른컬럼은 집계 함수와 함께 조회하는 것이 불가능하다.

	SUM(), AVG(), COUNT(), MAX(), MIN()

3) GROUP BY, HAVING 절
	GROUP BY 절 : 특정 컬럼에 있는 값을 그룹으로 묶어서 데이터를 집계한다.
	HAVING절 : GROUP BY 절 다음에 사용할 수 있으며, SELECT 의 WHERE 절처럼 조건을 걸어준다.

	WHERE 절 HAVING절 차이
	WHERE 절 : 쿼리문 전체에 조건을 걸어서 데이터를 가져오고 집계함수 사용 불가능하다.
	HAVING절 : 데이터를 GROUP BY로 묶어주는데 묶은 데이터에 대한 조건이 필요시 사용하고, 집계함수 사용 가능하다.

	SELECT 컬럼명
	FROM 테이블명
	WHERE 조건
	GROUP BY 컬럼명
	HAVING HROUP BY 에 대한 조건 OR 집계함수
	ORDER BY 컬럼명 ASC OR DESC;

1. 서브쿼리
   하나의 쿼리 내에 작성하는 또 다른 쿼리
   쿼리를 구분할 때 겉에 있는 쿼리를 메인쿼리, 내부에 있는 쿼리를 서브쿼리라고 한다

   FROM절 : IN LINE VIEW
   SELECT절 : SCALAR 서브쿼리
   WHERE절 : SUB QUERY
   
DDL : DROP, CREATE, ALTER, TRUNCATE
DML : SELECT, INSERT, UPDATE, DELETE

SELECT절의 서브쿼리 : 조회한 결과에서 다시 조회하는 서브쿼리

FROM절의 서브쿼리 : 테이블의 데이터를 서브쿼리로 조회하는 것을 의미

WHERE절의 서브쿼리 : 조건을 설정하기 위해 사용되는 서브쿼리
