복습 -> 서브쿼리 -> join

0. 복습
1) SQL 실행순서
   FROM > WHERE > GROUP BY > HAVING > SELECT > ORDER BY

2) SQL 쿼리문
   SELECT 컬럼명1 AS "별칭", 컬럼명2 별칭, 컬럼명3 "별칭 들"
   FROM 테이블명 t
   WHERE 조건
   GROUP BY 컬럼명
   HAVING 그룹에 대한 조건
   ORDER BY 컬럼명 정렬기준[ASC, DESC];

3) ROWNUM
   결과 행 앞에 1부터 1씩 증가하는 시퀀스를 붙여준다
   컬럼처럼 사용할 수 있다

1. 서브쿼리
	   하나의 쿼리 내에 작성하는 또 다른 쿼리

1) SELECT 절에서의 서브쿼리(SCALAR 서브쿼리)
	   조회한 결과에서 다시 조회하는 서브쿼리   

	 SELECT   (SELECT
      	FROM
      	)
   	FROM   
   	WHERE

2) FROM 절에서의 서브쿼리(IN LINE VIEW)
	   테이블의 데이터를 서브쿼리로 조회

	   SELECT 
	   FROM (SELECT
	      FROM
	      )
	   WHERE

3) WHERE 절에서의 서브쿼리
	   조건을 설정하기 위해 사용되는 서브쿼리

	   SELECT
	   FROM
	   WHERE (SELECT
	      FROM
	      WHERE )


2. join
   두 개 이상의 테이블을 연결하여 데이터를 검색하는 방법
   여러 테이블에 흩어져있는 정보 중 사용자가 필요한 정보만 가져와서 가상의 테이블처럼 만들고 결과를 보여주는 것

   정규화를 통해 조회할 테이블이 너무 많이 쪼개져 있으면 작업이 불편하기 때문에
   입력, 수정, 삭제의 성능을 향상시키기 위해서 join을 통해 합친 후 사용한다

1) 조인의 종류
   내부조인   공통으로 존재하는 컬럼의 값이 같은 경우를 추출하는 기법
   외부조인   왼쪽 외부조인   : 왼쪽 테이블의 모든 데이터와 오른쪽 테이블의 동일 데이터를 추출하는 기법
      오른쪽 외부조인   : 오른쪽 테이블의 모든 데이터와 왼쪽 테이블의 동일 데이터를 추출하는 기법
      완전 외부조인   : 양쪽의 모든 데이터를 추출하는 기법
   교차조인   조인조건이 없는 모든 데이터 조합을 추출하는 기법
   셀프조인   자기 자신에게 별칭을 지정한 후 다시 조인하는 기법

2) 내부조인(INNER JOIN)
   조건이 정확히 일치하는 값만 합쳐서 조회

문법
   SELECT A.컬럼명1, A.컬럼명2, B.컬럼명1, B.컬럼명2
   FROM 테이블명1 A INNER JOIN 테이블명2 B
   ON [조인조건] A.컬럼명 = B.컬럼명
   [WHERE 검색조건];
